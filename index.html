前端工程師個人網站主視覺占位區

這裡是個人品牌網站的 Hero 區主視覺，未來會放上一句話自我介紹、核心職稱（例如 Frontend Developer）、以及引導訪客瀏覽作品或下載履歷的行動呼籲。

目前這裡是示意用的占位內容，協助你在 Phase 1 先看見版型結構。正式上線前，會用 Section SDD 詳細描述：「主標、副標、關鍵技能標籤、按鈕配置與文字」，產生一整段新的 Hero 區程式碼。

給之後的開發者：正式內容產生後，請直接將新的 <section id="hero" ...>...</section> 整段貼在本段註解 SECTION:HERO-START / SECTION:HERO-END 之間，完全覆蓋現在這一區。

瀏覽作品集 ↗ 預約合作洽談

之後會用 Section SDD 的 Prompt 產生新的 Hero 區排版與文案，請記得只要「整段取代這裡的 section」，不用逐行手動修改。

開發者說明 · Hero 區 HTML 插槽：
之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 section#hero.section-hero 程式碼整段貼在 SECTION:HERO-START 和 SECTION:HERO-END 兩個註解中間，完全取代現在這一區。

Hero 區示意形象照占位：之後可改成個人照片或與前端開發相關的視覺。

ABOUT

關於我（之後會用 Section SDD 替換）

這裡目前是 About 區的占位內容，用來預留未來介紹個人背景、工作經歷、專長領域與側寫故事的版位。正式文案與版面會在下一階段透過 Section SDD 的 Prompt 產生。

在 Section SDD 階段，你可以描述希望呈現的層次（例如「一句話摘要」、「時間軸」、「價值觀與合作風格」），系統會輸出一整段新的 <section id="about" class="section section-about">...</section> 結構供你貼回來。

建議在正式版中放入 2～3 段簡潔的段落文字，讓訪客快速理解你是誰、擅長什麼、目前關注的技術或領域。

開發者說明 · About 區 HTML 插槽：
之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 section#about.section-about 程式碼整段貼在 SECTION:ABOUT-START 和 SECTION:ABOUT-END 兩個註解中間，完全取代現在這一區。

這段區塊主要用來提醒未來的你：「不需要在現有占位文字上做微調」，而是直接用新的 section 結構覆蓋，能避免 HTML 漏改或 class 遺失。

SKILLS

技能總覽（之後會用 Section SDD 替換）

這裡目前是 Skills 區的占位內容，未來會用結構化的方式呈現前端技術堆疊，例如語言、框架、工具與流程相關技能（版本控制、測試、部署等）。

你可以在 Section SDD 中指定想要的呈現形式，例如「Tag 雲」、「Skill 卡片等級」、「熟悉度條列」或「角色導向（Frontend / Fullstack / UI 協作）」等。

  • 前端技術堆疊占位
    之後會在這裡列出 HTML / CSS / JavaScript / 前端框架等核心技能與使用情境。
  • 工具與流程占位
    可放版本控制、建構流程、打包工具、測試框架、Lint 規則與 CI/CD 流程等資訊。
  • 溝通與協作能力占位
    若希望強調跨部門協作、UI/UX 對接、需求拆解與文件撰寫，也可以在這裡補充。

之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 section#skills.section-skills 程式碼整段貼在 SECTION:SKILLS-START 和 SECTION:SKILLS-END 兩個註解中間，完全取代現在這一區。
PROJECTS

作品集展示（之後會用 Section SDD 替換）

這裡目前是 Projects 區的占位內容，未來會放上實際專案卡片，例如個人 Side Project、公司專案片段、開源貢獻或教學示範作品等。

在 Section SDD 階段，你可以描述專案類型、重點技術、負責範圍與成效，系統會輸出對應的卡片結構與文案，讓作品集更有故事性。

  • 作品卡片占位（一）

    未來可在這裡放入某個代表性專案的名稱、簡短介紹與負責內容，讓訪客一眼看出你擅長解決的問題與使用的技術。

    之後會用 Section SDD 的 Prompt 產生正式的卡片結構與內容，例如 Demo 連結、GitHub 連結與技術標籤等。

  • 作品卡片占位（二）

    你也可以選擇只展示幾個最具代表性的專案，並用較多文字說明背景脈絡、決策過程與技術取捨，強調「如何思考」。

    產出的新 section 將直接覆蓋這一區，請在更新時整段貼上，而不是局部修改現有 HTML。

開發者說明 · Projects 區 HTML 插槽：
之後會用 Section SDD 的 Prompt 產生新的 section，請把新的 section#projects.section-projects 程式碼整段貼在 SECTION:PROJECTS-START 和 SECTION:PROJECTS-END 兩個註解中間，完全取代現在這一區。
